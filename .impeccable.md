# .impeccable.md — Design Context

## Design Context

### Users
Смешанная аудитория: разработчики Telegram-ботов **и** их клиенты (нетехнические пользователи). Инструмент открывается преимущественно в **Telegram Mini App на мобильном** и в **Telegram Desktop**. Пользователи запускают его в рабочем контексте — строят клавиатуры, формируют API-запросы, копируют JSON в бота. Сценарий: быстро сделать нужное и закрыть.

### Brand Personality
Три слова: **Удобно. Дружелюбно. Понятно.**

Интерфейс должен ощущаться как хорошо отточенный профессиональный инструмент — не как хакерский терминал. Ближайшая аналогия: **Postman**, **Linear**, или внутренний dev-tool от компании с хорошим дизайн-отделом. Доступный, без барьеров входа.

### Aesthetic Direction
**Отказ от текущей cyberpunk/neon эстетики** (Orbitron, cyan glows, scanlines, агрессивная магента). Она конфликтует с принципом «понятно и дружелюбно», особенно для нетехнических пользователей.

**Новое направление: Clean Dark Tool**
- Тёмная тема (соответствует контексту Telegram, удобна в любое время суток)
- Мягкая, сдержанная палитра — тёмный индиго/синий-серый, без кислотного неона
- Акцент — один, спокойный (синий или фиолетовый средней насыщенности, не кричащий)
- Типографика — читаемая и иерархичная: одна чёткая display-пара + legible mono для кода
- Углы — чуть больше скруглений (4–6px), меньше острых граней
- Без сканлайн-оверлеев, без glow-пульсаций, без Orbitron everywhere

**Антиреференс**: текущий стиль проекта — агрессивный кибerpunk с `#00f5ff` glow.  
**Референс-настроение**: тёмный Postman, Raycast, Telegram's own dark UI, Linear в тёмной теме.

### Design Principles

1. **Функция главнее атмосферы** — интерфейс инструмента, а не витрина. Каждый декоративный элемент должен зарабатывать своё место; если убрать — ничего не потеряется, убрать.

2. **Мобильный сначала** — минимальные touch-targets 44×44px, читаемость 13–14px без зума, поля достаточного размера для пальца. Telegram Mini App — главная точка входа.

3. **Один акцент — максимум** — не cyan + magenta + green + red все сразу. Один фирменный цвет для интерактивных элементов, семантические цвета (success/danger) — только по делу.

4. **Иерархия через пространство, не через свечение** — вместо box-shadow glow для выделения — отступы, вес шрифта, контраст. Glow — только в крайнем случае и не анимированный.

5. **Нетехнический пользователь должен понять за 10 секунд** — лейблы читаемы, структура очевидна, нет аббревиатур без расшифровки.
